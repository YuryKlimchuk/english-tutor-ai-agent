# Session 2025-06-21 (v2)

## Finalized architecture

### Package convention
```
module/
├── api/                    # ID records, public contracts
├── application/
│   └── config/properties/  # @ConfigurationProperties implementations
├── domain/
│   └── <entity>/           # per-entity package
│       ├── Entity.java
│       ├── EntityRepository.java
│       ├── *Id.java
│       ├── *SpecificationConfig.java (interface)
│       └── *Specifications.java
└── infrastructure/         # JPA, messaging later
```

### Dictionary module (current)
```
dictionary/
├── api/DictionaryId.java
├── application/config/properties/
│   └── WordSpecificationSpringProperties.java  # implements domain interface
├── domain/
│   ├── dictionary/
│   │   ├── Dictionary.java                    # @AggregateRoot, List<Word>
│   │   └── DictionaryRepository.java
│   └── word/
│       ├── Word.java                          # @Entity, final class
│       ├── WordId.java                        # record
│       ├── PartOfSpeech.java                  # NOUN, VERB, ADJECTIVE
│       ├── WordSpecificationConfig.java        # interface (domain contract)
│       └── WordSpecifications.java            # instance class, config via constructor
└── package-info.java
```

### SpecificationConfig pattern
- **Domain**: `WordSpecificationConfig` — interface (what domain needs)
- **Application**: `WordSpecificationSpringProperties` — @ConfigurationProperties, implements interface
- **Specifications**: depend on interface, not Spring — clean DDD
- Spring binding via `@Accessors(fluent = true)` to match interface methods

### Word.java
- `final class`, `@RequiredArgsConstructor`
- `@Identity WordId id` + immutable fields: value, translations
- Mutable: partOfSpeech, targetRepetitions, currentRepetitions
- Status derived from current vs target (not stored)
