## Architecture conventions
1. Per-entity packages: `domain/<entity>/`, `api/<entity>/`
2. API: `*Id`, `*Command`, `*` (port) in `api/<entity>/`
3. Domain: Entity, Repository, Factory, Specs, Config (interface), Exception hierarchy
4. Error: `DomainException` → `<Entity>DomainException` → specific
5. Config: interface in domain, `@ConfigurationProperties` in application
6. `@ApplicationModule(allowedDependencies = "...")` — explicit module boundaries
7. Mermaid sequence diagrams in module docs
