# Session 2025-06-20 (v4 — deep architecture)

## Архитектурные решения

### Отношения bounded context'ов (финал):
```
Teacher ──owns──→ Student          (учитель создаёт учеников)
Teacher ──starts──→ Lesson         (учитель начинает урок)
Lesson ──with──→ Student           (урок с конкретным учеником)
Lesson ──feeds──→ Dictionary       (слова с урока попадают в словарь)
Dictionary ──Student owns──→       (студент ссылается на словарь)
Teacher ──generates──→ Exercise    (учитель генерирует упражнение)
Exercise ──uses──→ Dictionary      (на основе словаря)
```

### Dictionary module (реализовано):
- **Dictionary** — @AggregateRoot, содержит `Set<WordId>` (ссылки по ID)
- **Word** — @Entity, только id (поля позже)
- **WordId** — record в `domain/model/`

### Student ↔ Dictionary (обсудили):
- Student хранит `DictionaryId`, Dictionary не знает про Student
- Создание: use case в **Teacher** модуле оркестрирует:
  1. `Dictionary.api` → создать словарь → `DictionaryId`
  2. `Student.create(id, dictionaryId, ...)`
  3. `studentRepository.save(student)`
