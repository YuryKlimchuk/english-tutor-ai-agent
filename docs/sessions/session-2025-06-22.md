### All conventions established
1. Package grouping: per-entity in both domain/ and api/
2. API: <Entity>Id, <Action>Command, <Action> (port) in api/<entity-lowercase>/
3. Use Case: port in api, impl in application/usecase/
4. Config: interface in domain, @ConfigurationProperties in application
5. Validation: Specification<T> from shared, instance class with config
6. Error handling: DomainException hierarchy per module
7. Factory: creates validated entity, config via constructor
8. Tests: domain tests without Spring, use stubs for repositories
