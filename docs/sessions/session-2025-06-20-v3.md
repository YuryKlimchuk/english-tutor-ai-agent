# Session 2025-06-20 (вечер v3 — reasoning)

## Обсудили архитектуру

### Отношения между bounded context'ами:
```
Teacher ──owns──→ Student          (учитель добавляет учеников)
Teacher ──starts──→ Lesson         (учитель начинает урок)
Lesson ──with──→ Student           (урок с конкретным учеником)
Lesson ──feeds──→ Dictionary       (слова с урока попадают в словарь)
Dictionary ──belongs to──→ Student (словарь принадлежит ученику)
Teacher ──generates──→ Exercise    (учитель генерирует упражнение)
Exercise ──uses──→ Dictionary      (на основе словаря ученика)
```

### Ключевые решения:
- Student — дочерний для Teacher, не имеет своей аутентификации
- Ученик не работает с агентом напрямую
- ТГ — это реализация, абстрагируемся (externalId вместо telegramId)

### Что каждый модуль должен уметь (черновик):
- **Teacher**: профиль, аутентификация (абстрактная), управление учениками
- **Student**: привязка к Teacher, без аутентификации
- **Lesson**: старт с Teacher+Student, сбор слов, завершение → слова в Dictionary
- **Dictionary**: хранение слов по темам/урокам, доступен Teacher'у
- **Exercise**: генерация агентом на основе Dictionary

### Открытые вопросы (на подумать):
1. Нужен ли Student'у прогресс (какие слова выучил)?
2. Lesson — полноценная сессия с началом/концом или просто «пачка слов за раз»?
